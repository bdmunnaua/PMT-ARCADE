# Security Policy

## Supported Versions

Only the latest stable version of PMT ARCADE is supported with security updates.

| Version | Supported |
|---------|-----------|
| 1.x (Latest) | ✅ Fully supported |
| Beta releases | ⚠️ Limited support |
| Older versions | ❌ Not supported |

## Reporting a Vulnerability

If you discover a security vulnerability in PMT ARCADE, please report it immediately.

### How to Report

1. **Do NOT** create a public GitHub issue for security vulnerabilities.
2. Send an email to: **admin@pmtarcade.com**
3. Include in your report:
   - Description of the vulnerability
   - Steps to reproduce
   - Potential impact
   - Any suggested fix (optional)

### What to Expect

- You will receive a response within **48 hours**
- We will investigate and confirm the vulnerability
- We will work on a fix and release an update
- Once fixed, we will acknowledge your contribution (if you wish)

### Responsible Disclosure

We follow responsible disclosure practices:
- Vulnerabilities will be fixed before public announcement
- We will credit researchers who report valid issues
- Please allow up to 30 days for a fix before public disclosure

## Security Best Practices for Users

- Always use a strong, unique password for your PMT ARCADE account
- Never share your private keys or seed phrases
- Only withdraw to wallets you control
- Verify the website URL is `https://pmtarcade.com` before logging in
